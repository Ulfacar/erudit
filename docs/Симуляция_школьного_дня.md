# Симуляция полного школьного дня в BilimOS

> Один обычный учебный день в школе, прожитый через платформу — по ролям, по часам,
> с реальными модулями и автоматическими реакциями ИИ (событие → действие).
> Всё описанное опирается на уже работающие части продукта. Demo-вход: пароль `erudit2025`.

---

## Таймлайн дня (вторник)

### 07:30 — До звонка
- **Завуч (`/today`, `/dashboard`)** открывает «Главную»: сводка по школе — успеваемость, посещаемость, нагрузка. **Агент-панель (`/agent`)** уже сформировала за ночь список инсайтов (детерминированные правила, без LLM): что требует внимания сегодня.
- **АХЧ (`/workspace/maintenance`)** видит заявку «не работает проектор в 5Б» — берёт в работу.
- **Кухня (`/workspace/kitchen`)** подтверждает меню на сегодня и количество порций.

### 08:00 — Первый звонок, посещаемость
- **Куратор (`/journal/attendance`)** отмечает класс: 22 присутствуют, 2 опоздали, 1 болеет.
- ⚙️ **Событие → действие:** если за месяц процент пропусков класса превышает среднешкольный на ≥5 п.п., **агент** поднимает сигнал «Пропуски выше нормы: класс 7Б — 18% при среднем 9%» со ссылкой на `/reports/attendance`. Завуч видит это в панели, не собирая отчёт руками.

### 08:15–13:00 — Уроки
- **Учитель (`/today`, `/grading`)** ведёт урок, выставляет оценки. Категории оценок имеют веса → средний балл считается взвешенно.
- Поставлена двойка ученику.
- ⚙️ **Событие → действие:** если средний балл ученика просел на ≥0.7 за 2 недели, **агент** формирует сигнал «Падение успеваемости: Иванов И., 7Б — −0.9 за 2 недели, сейчас 3.2. Рекомендуем разговор с куратором» → `/students/{id}`. При падении ≥1.2 сигнал помечается как **urgent**.
- **Учитель** готовит **поурочный план / ИИ-презентацию** к следующему уроку (`/lesson-plans`, `/presentations`) — генерация за минуты (в demo-режиме на готовых данных; в бою — через будущую систему токенов Урмата).
- **Ученик (`/diary`)** видит новую оценку и домашнее задание. **ИИ-тьютор** (demo) помогает разобрать тему, наводя на ответ, а не выдавая готовый.

### 10:30 — Перевод между группами (новый модуль)
- **Куратор (`/group-transfers`)** запрашивает перевод ученика из группы «Англ-1» в «Англ-2». Принимающий учитель **одобряет**. Система:
  - проверяет лимит **5 переводов на класс в месяц** (жёстко по ТЗ),
  - физически перемещает ученика между группами,
  - создаёт **срочный вопрос** для завуча/директора как уведомление.

### 11:00 — Психологическая служба (eSPSMS)
- **Психолог (`/psychologist`)** ведёт кейс, заполняет сессию (DAP), фиксирует уровень риска.
- ⚙️ **Fail-closed приватность:** данные перед уходом в облако обезличиваются; при сбое обезличивания запрос **не уходит** (закрыто по умолчанию).
- **Красный сигнал** (буллинг/риск) → **авто-эскалация** к старшему психологу и координатору безопасности (`/safeguarding`). Учитель про себя свои L2/L3-дескрипторы **не видит** — только L1.

### 11:30 — Происшествие
- **Учитель/секретарь (`/incidents`)** регистрирует происшествие (конфликт на перемене): тип, серьёзность, статус. Завуч видит в `/urgent-issues`, ведёт до «решено».

### 12:00 — Медкабинет и логопед
- **Врач (`/workspace/medical`)** фиксирует визит и рекомендацию.
- **Логопед (`/workspace/speech`)** отмечает занятие и прогресс ученика.

### 13:00 — Приёмная (CRM)
- **Секретарь (`/admission`)** ведёт воронку: заявка → встреча → зачисление. Новая семья записалась на собеседование.
- ⚙️ **Событие → действие:** заявки без движения >7 дней → **агент** напоминает «Заявки без движения: 3 — семьи могут уйти в другую школу» → `/admission`.

### 14:00 — Финансы
- **Родитель (`/finance`)** оплачивает обучение прямо на платформе (PWA + пуш-уведомление о зачислении).
- **Бухгалтер (`/workspace/accounting`)** видит счета и оплаты.
- ⚙️ **Событие → действие (деньги):**
  - агрегатная задолженность → «Задолженность: N сом, M учеников» → `/workspace/accounting` (без публичного позора конкретных семей);
  - **пени** за просрочку начисляются автоматически (0.1%/день) → отдельный сигнал;
  - **колл-центр (`/call-center`)** получает список задолжников для обзвона.

### 15:00 — Управление и кадры
- **HR (`/hr`, `/staff`)** ведёт сотрудников, контракты, вакансии, отпуска, зарплаты.
- **Завуч** при необходимости делает **передачу нагрузки** учителя (`/teachers/workload`) — напр., уход в декрет, с read-only историей.
- **Секретарь** оформляет **заявление** об отсутствии или **отчисление** (`/applications`, `/withdrawals`).

### 16:00 — Импорт/экспорт и документы
- **Секретарь (`/import-export`)** массово загружает учеников из CSV (классы создаются автоматически) или выгружает список в Excel.
- **HR/секретарь (`/documents`)** регистрирует документы (приказ, договор, СНИЛС) с привязкой к ученику/сотруднику.

### 17:00 — Конец дня: аналитика и картина
- **Аналитик/Завуч (`/analytics`)** смотрит живые графики: средняя успеваемость по классам, посещаемость за неделю, распределение оценок, рейтинг учителей, отстающие ученики — всё из реальных данных.
- **Родитель** получает (в перспективе Фазы 2) еженедельный **ИИ-дайджест** о ребёнке.
- **Учредитель (`admin`)** видит школу как бизнес: набор, отток, деньги — прозрачность в любой момент.

## Агентная цепочка дня (событие → действие), сводно

Это сердце продукта — система не просто хранит данные, а **действует на опережение**.
Реальные правила (`src/shared/lib/ai/insights.ts`, детерминированные, без LLM):

| Событие в течение дня | Автоматическая реакция системы |
|---|---|
| Средний балл ученика просел ≥0.7 за 2 недели | Сигнал куратору (urgent при ≥1.2) |
| Пропуски класса ≥5 п.п. выше школьных | Сигнал «выяснить причину» завучу |
| Счета не оплачены | Сводка долга бухгалтерии + список колл-центру |
| Просрочка оплаты | Авто-начисление пени 0.1%/день |
| Заявка в приёмной стоит >7 дней | Напоминание «семья может уйти» |
| Красный психологический сигнал | Fail-closed эскалация в службу безопасности |
| Перевод между группами одобрен | Уведомление завучу + контроль лимита 5/мес |

---

## Что эта симуляция показывает (для встречи и для Эмира)

1. **Это операционная система школы, а не электронный журнал** — закрыт весь день каждой роли.
2. **Проактивность «событие → действие»** работает на реальных данных уже сейчас.
3. **Готовность к пилоту:** ядро, финансы, психология, приёмная, кадры, аналитика — рабочие.
4. **Что ждёт включения:** ИИ-ассистент в боевом режиме — через систему токенов Урмата (сейчас demo).

## Живой прогон (реально выполнен)

Есть исполняемый скрипт `scripts/sim-school-day.ts` — прогоняет этот сценарий через Prisma
против **локальной** БД (`localhost:5433`), создаёт события дня и запускает реальную агентную
логику. Идемпотентный (метки `sim:`), прод не трогает.

```
npx tsx scripts/sim-school-day.ts
```

**Фактический вывод прогона (класс 8а, демо-данные):**
```
08:00  Посещаемость: 6 присут., 1 опозд., 1 отсут.
09:10  Оценки по «Кыргызский язык»: 2, 5, 4, 3, 5 — двойка у Бекмуратова Бегимай
10:00  Задано ДЗ (срок +2 дня)
⚙️ АГЕНТ: 17 неуд. оценок в 8а → сигнал куратору
⚙️ АГЕНТ: посещаемость 8а 75% → ниже нормы, сигнал завучу
⚙️ АГЕНТ: 9 учеников с неоплаченными счетами → бухгалтерии напоминания, колл-центру обзвон
⚙️ АГЕНТ: 10 просроченных счетов → авто-начисление пени 0.1%/день
17:00  Средний балл 8а: 4.07 (230 оценок) · в школе 162 ученика, 16 педагогов
```

Подтверждает: **событие → действие работает на реальных данных, а не на словах.**

### Полный визуальный прогон (по экранам, когда нужно для показа)
1. Поднять БД: `docker compose up -d`; приложение: `npm run dev` (`--webpack` из-за кир. пути).
2. Слепое QA по ролям: скилл `journey-qa`, либо Playwright-smoke: `npx playwright test smoke-pages`.
