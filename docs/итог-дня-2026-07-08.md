## 🧠 Терминал 1 — Модуль психолога eSPSMS v2.0 (ТЗ Эмира, 22 правки)

**Статус: ВСЁ ТЗ выполнено (22/22 CR) и ЗАДЕПЛОЕНО на bilimos.kg.**
Реализовано дельтой поверх существующего модуля (keep-in-place, реюз-первый). 20 коммитов.

### Что сделано по волнам
**Волна 1 — фундамент (критично):**
- **CR-001/002/003** — 6-этапный цикл кейса (Оценка→Диагностика→ИПС→Интервенция→Обзор→Закрытие) вместо «курс/фаза»; степпер; «психологическая интервенция» (серия сессий, привязана к ИПС); ИПС как версионируемая сущность. Терминология «курс/фаза/раунд» вырезана (grep=0). Гейты на API: нельзя начать интервенцию без утверждённого ИПС, нельзя вторую активную.
- **CR-015** — матрица видимости ФИО/код **на уровне API** + новая роль `psy_coordinator` (Координатор ПС). Непривилегированные роли видят код участника (`psyCode` У-####), а не ФИО. Старший психолог — код; координатор ПС и ведущий психолог — ФИО. e2e-автотест на все роли.

**Волна 2 — под сентябрьский дедлайн:**
- **CR-004/005** — массовый онлайн-скрининг: кампании, охват в %, прохождение учеником (/screening), авто-пометка «группа риска», каталог методик по ступеням (9 методик 1-4/5-9/10-11).
- **CR-008** — блок «Приём»: экспресс-тест поступающих психологом, enum-заключение (рекомендован/не рекомендован/перенаправлен), событие в приёмную, авто-привязка к профилю при зачислении.

**Волна 3:**
- **CR-003/022** — ИПС SMART-цели (5 полей-подсказок, срок из фикс-списка 2н/3н/4н/3м/6м, 1–3 направления на цель).
- **CR-009** — главный экран психолога: блок экстренных обращений, индикатор «!» истекающей документации, кнопка «Приём».
- **CR-013** — feedback-loop оценки ИИ-черновика DAP (звёзды 1–5), сводка средней оценки координатору.
- **CR-018** — переоткрытие закрытого кейса с нового цикла (этап 2/3) + новая версия ИПС.

**Волна 4:**
- **CR-014** — трёхуровневая отчётность (психолог→свои, координатор→все+ИИ, админ→анонимно); ГорОНО исключён (в коде и не было).
- **CR-007/021** — конструктор методик открыт всем психологам (свои приватны до публикации старшим; базовые нередактируемы, только копия); номера версий в UI.
- **CR-010/011** — кабинеты Родители (ссылки на детей-сиблингов) и Учителя (история консультаций/тренингов + тип «онлайн-модуль»).
- **CR-020** — fail-closed: для красных кейсов внешний ИИ запрещён всегда (даже после обезличивания).
- **CR-016/012** — safeguarding только инициалы (без перехода к карточке) и DAP — подтверждены (уже были).
- **CR-017** — peer-консилиум убран по ТЗ Эмира (RLS оставляет только свои кейсы; старший психолог видит все read-only).

### Осознанные решения
- **CR-017**: сначала оставляли (правило «не убирай, демо показывают»), потом по решению Алана убрали — это правка заказчика, не наше урезание; замена из самого CR-017 (обзор старшего психолога) сохраняет демо-историю.
- **CR-006 (CogniFit) и CR-019 (политика аудио)** — doc-only по ТЗ, без кода в v2.0: нужна справка/решение владельца продукта. Дозапросить у Эмира.
- **e2e матрицы ФИО** написан, но не прогнан локально (нужен seeded-сервер) — гонять при QA.

### Процесс
- Fable-ревью поймал и мы закрыли **3 приватности-бага**: утечка ФИО родителя/учителя (`subjectName`), риск-статус ученику в network-табе, стирание заметки психолога пустым полем.

## 🚀 Деплой
- `psy-espsms-v2 → main` (merge чист) → `git push erudit main` (`7f8c5c0..ae3ced6`).
- Coolify: сборка ручным триггером через API (авто-вебхук не поднимает), deployment `lpl1h3an55rvrvh6hjn9ir4i`, ~15-20 мин, `predeploy.mjs` делает `db push` новых psy-таблиц + сиды.
