## 1. Модуль психолога (eSPSMS) — 100% ТЗ
- **Кейсы + RLS** (психолог видит только свои), коллабораторы/консилиум, уровни риска,
  красный → текстовое обоснование + слепой safeguarding-алерт.
- **Voice-to-DAP**: микрофон (Web Speech), **серверное обезличивание ФИО→`[УЧЕНИК_1]`** до
  облачного LLM, раскладка DAP, оффлайн-черновик (IndexedDB), anti-hallucination галочка.
- **Vision проективных тестов**: фото → клиентский блюр подписи → **оригинал в приватный
  MinIO-сейф**, заблюренное в Vision → черновик → верификация.
- **Конструктор методик** с версионированием + mapping-rule (патч «аналитического коллапса»),
  типы вопросов: шкала/текст/симптом/**файл**.
- **Safeguarding**: слепые алерты координатору, закрытый контур (инициалы+причина), «взять в работу».
- **Дашборд директора** — анонимные агрегаты (без ФИО/текстов).
- **Доводка**: курс из 6 сессий + исход (improved/repeat/referred) + эскалация к психиатру +
  уведомление старшему; качественное «до/после»; **PRE-тест при поступлении** (хук приёмки).
- **OMR бланков** (камера→баллы) + **калькулятор шкалы** (ответы→сумма→график динамики).
- Роли: `psychologist`, `senior_psychologist`, `safeguarding_lead`. Маршруты `/psychologist*`,
  `/safeguarding`. API `/api/v1/psy/*`.
