# Ответы Эмира по College Consulting (получено 08.07.2026)

Источник: `Вопросы_Эмиру_College_Consulting_BilimOS.pages` (Telegram, ответы вписаны Эмиром).
Расшифровка из iWork-архива. Ниже — решения, влияющие на доработку модуля.

## A. Даты и дедлайны
**1. Какие даты дедлайна разрешать?**
→ **Разрешить любую будущую дату** (запрещать только прошедшие).
Причина: дедлайны вузов часто попадают на следующий цикл (ноябрь–январь след. года), летом такой не внести.
*Работа:* поправить валидацию (сейчас — только текущий уч. год) и клиентское ограничение инпута.

## Б. Матрица «Цели ученика vs Ожидания родителей»
**2. Порог бюджета для «на грани» (жёлтый).**
→ **НЕ фиксированный $10 000.** Порог у каждой семьи свой, обговаривается индивидуально.
«Должна быть возможность внесения порога для каждой семьи индивидуально.»
*Работа:* убрать хардкод $10k, добавить поле порога на семью/профиль, вводимое вручную.

**3. Правила цвета конфликта — верны?**
→ **Всё верно.** (ЗЕЛЁНЫЙ/ЖЁЛТЫЙ/КРАСНЫЙ логика ок.) Изменений нет.

## В. Экзамены и академика
**4. Типы внешних экзаменов.**
→ **Всё верно**, но **добавить китайские экзамены** (напр. HSK) если можно, и дать **СС возможность добавлять свои** типы.
«Давай базовые ведём такие, и какие-то чтобы СС мог добавить сам.»
*Работа:* расширяемый справочник экзаменов (базовые + пользовательские).

**5. Проверка GPA против требований вуза — откуда брать требования?**
→ **Консультант вводит требуемый GPA вручную для каждого вуза.**
«При добавлении вуза он вводит перечень документов, требования и прочее.»
*Работа:* поля на сущности вуза: требуемый GPA, перечень документов, требования/заметки. Предупреждение GPA считать от введённого вручную значения. Справочник вузов с базой требований — НЕ нужен.

## Г. Документы и рекомендации
**6. Авто-чек-лист документов по стране/вузу — нужен?**
→ **Нет.** Типовых шаблонов нет, **СС заполняет сам** (связано с №5 — доки вводятся вручную на вуз).
*Работа:* НЕ строить авто-шаблоны. Ручной перечень доков на вуз (см. №5).

**7. Как учитель сдаёт рекомендательное письмо?**
→ **Оставить как проще** — ссылка на файл (как сейчас). Изменений нет.

**8. Проверка эссе через ИИ — сейчас?**
→ **Позже, не в этом этапе.**
*Работа:* Tier C (AI-эссе с обезличиванием) — ОТЛОЖЕНО, не делаем сейчас.

## Д. Отчёты, доступ, выпуск
**9. Годовой отчёт директору — данных достаточно?**
→ **Достаточно текущего.** Изменений нет.

**10. Достижение при поступлении — кому видно?**
→ **Видно всем в профиле** ученика (общая лента достижений).
*Работа:* проверить, что созданное Tier A достижение публично, не приватно консультанту.

**11. Доступ к матрице конфликтов и черновикам эссе — верно?**
→ **Всё верно.** (Консультант+суперадмин видят; директор — отчёт без матрицы.) Изменений нет.

**12. Общий канбан ВСЕХ заявок на дашборде — нужен?**
→ **НЕТ.** Достаточно канбана внутри карточки ученика (уже есть).
*Работа:* глобальный канбан-дашборд НЕ строить. Отменяется.

## Итог — реальный остаток работ (Tier B/C переопределён)
Делаем (все — лёгкие конфиг-правки, без тяжёлых фич):
1. **№1** дедлайн: разрешить любую будущую дату.
2. **№2** порог бюджета: индивидуальный на семью, ручной ввод (убрать $10k).
3. **№4** экзамены: + китайские, + пользовательские типы.
4. **№5+6** сущность вуза: ручные поля требуемый GPA + перечень доков + требования; предупреждение GPA от них.
5. **№10** достижение: подтвердить публичную видимость.
