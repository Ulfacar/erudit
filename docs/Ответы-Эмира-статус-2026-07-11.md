## Открытые вопросы (Часть 2)

- **2.1 College Consulting — справочник вузов.** Решение: **делаем базовый справочник вузов** — консультант один раз заводит вузы (стоимость обучения, специальности, требуемые документы), при выборе вуза для ученика данные подтягиваются автоматически. (Это разворот ответа №5 от 08.07 «вручную» — приоритетнее старого.)
- **2.2 Психолог CR-006 (CogniFit).** Откладываем. Интеграцию делаем позже, сейчас не приоритет.
- **2.3 Психолог CR-009 (макеты главного экрана).** Да — запросим у координатора ПС оригиналы схем главного экрана.
- **2.4 Психолог CR-019 (аудиозаписи сессий).** Утверждено: (а) хранить **на сервере школы в БД**; (б) срок хранения — **2 года**; (в) доступ — **только психологи** (автор записи + психологи с ролью).
- **2.5 G7 (шаблон трудового договора).** Да, шаблон есть — пришлёт.
- **2.6 Олимпиада (6 вопросов Волны 3).** Согласен пройтись на встрече; просит прислать список 6 вопросов заранее.

## На подтверждение (Часть 3)

- **CR-020 — внешний ИИ после обезличивания.** ✅ **Ответ Эмира:** дефолт в целом подтверждаем, но нужен **баланс между ИИ и ручной проверкой** — **НЕ полный запрет ИИ для красных, а ИИ как вспомогательный инструмент с обязательной ручной проверкой психологом**.
  - **Реализация (13.07):** для красных кейсов DAP-**текст** уходит в облако **обезличенным** (ФИО/ДР/телефоны заменяются маркерами до отправки), результат помечается баннером «требует обязательной ручной проверки психолога». **Картинки** проективных тестов/бланков (vision/omr) для красных **остаются локально** (обезличить изображение нельзя). Строгий режим приватности при остаточном PII по-прежнему держит текст локально (fail-closed сохранён на этом пути).
- **CR-021 — версионирование методик.** ✅ Подтверждаем: старые результаты при обновлении методики не пересчитываются (хранятся с версией).

## К запуску Intellect на своём сервере (Часть 5)

- Заливку реальных данных начинаем **как только сервер готов**. Ждём готовности инстанса и аудита безопасности/приватности с нашей стороны.
