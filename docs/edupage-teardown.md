# EduPage — функциональный инвентарь и план паритета для Bilim OS

> **Решение основателя (2026-05-30):** строим к полному функциональному паритету с EduPage.
> **Исключаем:** платёжные шлюзы / банковский эквайринг, Face ID / биометрию / турникеты-СКУД.
> **Метод:** competitive teardown — каталог *возможностей* конкурента (что система умеет), без копирования их кода, текстов, контента и дизайна. Это research-артефакт и build-roadmap, не обязательство «всё к понедельнику».
> **Near-term (демо понедельник):** учительский cockpit (главная «Сегодня» → урок → быстрые оценки баллами с весами). Остальное — фазами.

Легенда: ✅ есть · 🟡 частично (есть база, надо довести) · ❌ нет · ⛔ исключено основателем.

## Фазовый план стройки (паритет без банков и биометрии)

**Фаза 0 — демо (понедельник):** учительский cockpit — главная «Сегодня» → урок → быстрые оценки баллами с весами; privacy учителя server-side; модерация гасится флагом. *Зависит от 2 решений основателя: вес на типе работы vs на каждой работе; подтверждение флага модерации.*

**Фаза 1 — «канцелярия» на нашей generic-инфре (createCrud/ResourcePage), недели:**
заявления-онлайн (отпросить → завуч), электронные согласия родителей, опросы/голосования, заказ питания (без оплаты), бронь кабинетов, бюро находок, материалы урока, сдача ДЗ онлайн (с загрузкой файлов). → закрывает большинство ❌ из §4,6,7,8,9.

**Фаза 2 — средние модули (недели-месяцы):**
банк тестов + онлайн-экзамен с авто-проверкой закрытых типов; полуавтомат расписания (ручное + валидатор конфликтов); push-уведомления родителям; CSV-импорт классов/учеников; PWA для родителя/ученика.

**Фаза 3 — «видение» / R&D (без жёстких дат):**
полноценный constraint-solver автосоставления расписания; OCR-проверка рукописных работ; нативные мобильные приложения; углублённая Кундолюк-интеграция (наш ключевой ров против EduPage).

**Исключено навсегда из scope (решение основателя):** платёжные шлюзы/банковский эквайринг, Face ID/биометрия, турникеты-СКУД. Софт-точки интеграции (приём события прохода, redirect на внешнего платёжного провайдера) — возможны позже через партнёра, без нашей разработки железа/PCI.

## Углублённый разбор функционала EduPage (что БЕРЁМ) — по их докам/приложению

Изучено по help.edupage.org и описанию приложения. Ниже — конкретные механики (возможности, своими словами), отмечено что берём (🟢), уже есть (✅), позже (🔵), не берём (⛔).

**Журнал учителя (ядро, наш текущий cockpit):**
- ✅🟢 Новая работа = новая колонка в журнале; у работы свой тип и вес; итог = взвешенное среднее. ← *это ровно то, что мы делаем в `/today`.*
- 🟢 **Тема урока прямо из расписания**: учитель к уроку записывает пройденную тему; следующую можно выбрать из учебного плана (КТП). ← добавить в карточку урока cockpit (поле «Тема», подтягивать из КТП).
- 🔵 Доп. баллы за активность на уроке прямо из журнала. ← мелочь, потом.
- 🟢 **Родитель/ученик «подписывает» новую оценку** (подтверждает, что видел). ← сильная EduPage-фича вовлечения родителей; добавить флаг «просмотрено родителем» на Grade.

**Посещаемость:**
- ✅ Учитель отмечает отсутствующих.
- 🟢 Куратор добавляет примечание к отсутствию; **родитель отправляет электронную записку об отсутствии с телефона**. ← заявление-«отпросить»/записка (пересекается с §7), берём.

**Домашние задания:**
- ✅🟢 Учитель задаёт ДЗ; ученик/родитель отмечает «выполнено». ← у нас ДЗ есть, добавить отметку «выполнено» (флаг) — дёшево.

**Расписание:** ✅ своё расписание + изменения по заменам на сегодня (у нас teacher-today это уже частично делает).

**Сообщения:** ✅🟢 написать учителю/классу/родителям; **групповое обсуждение с классом или со всеми родителями класса**. ← у нас чаты есть, добавить «рассылка классу/родителям класса».

**Тесты/e-learning:** 🔵 создание интерактивных тестов, авто-сбор и авто-оценка результатов. ← Фаза 2 (главная дыра), берём, но не сейчас.

**Столовая:** 🟢 недельное меню + **заказ/отмена обеда по дням** (без оплаты ⛔). ← Фаза 1, берём показ+заказ, оплату нет.

**Заявления / sign-up:** 🟢 электронные заявления (обработка с маршрутом), запись на родительские собрания, опросы. ← Фаза 1, берём.

**Прочее:** 🟢 бюро находок, кружки/секции (есть), сертификаты/справки (печать PDF) 🔵, переход на новый учебный год 🔵, права/роли ✅, отчёты ✅, конфиденц. записи об ученике ✅ (дескрипторы/мед). ⛔ турникеты/Face ID, ⛔ платёжный шлюз.

**Вывод — приоритет «что берём» прямо сейчас (в порядке):**
1. Допилить cockpit учителя: **тема урока** (+связь с КТП), **отметка «выполнено» у ДЗ**, **подпись оценки родителем**.
2. **Чистка меню по ролям** (учитель видит ~10 essentials — как EduPage role-based menu).
3. Фаза 1 (generic-инфра): заявления-онлайн + электронная записка об отсутствии, опросы, заказ меню, согласия, бюро находок.
4. Фаза 2: модуль тестов с авто-проверкой.

*Источники изучения: help.edupage.org (категории и модули), описание приложения EduPage (mobile.edupage.org / сторы) — функциональные возможности, без копирования их контента.*
